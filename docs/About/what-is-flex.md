# What is Flex?

Flex is a separate standard from [sUNC](https://docs.sunc.su/About), designed as its "little brother" to provide a space for Quality of Life (QoL) functions that do not meet sUNC's strict inclusion criteria.

## Purpose and Design

While sUNC rigorously excludes functions that can be implemented in Lua or are deemed unnecessary, Flex offers a compromise by allowing select convenient functions that enhance user experience without compromising on testing quality. This approach addresses numerous community requests for additional functions while maintaining the integrity of sUNC as a lean, essential standard.

Flex **remains strict in its own right**, carefully curating only a small set of valuable QoL functions rather than adding numerous bloated features. These functions still undergo proper testing.

## Relation to sUNC

Flex exists independently from sUNC to prevent the core standard from being overwhelmed and/or bloated by convenience-focused additions. Functions in Flex are thoroughly documented and tested, providing users with reliable QoL enhancements that sUNC intentionally omits. This separation allows sUNC to remain focused on essential functions while Flex caters to somewhat practical needs.
